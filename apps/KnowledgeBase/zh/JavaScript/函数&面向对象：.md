# 函数&面向对象

### 函数

函数本质是对象，可以被赋值，作为参数传递，作为返回值，挂载属性；

#### arguments和剩余参数

arguments：函数内部自带的一个伪数组，存储传入函数的所有参数；

```
function fn() {
  console.log(arguments);
}
fn(1, 2, 3);

//输出
{
  0: 1,
  1: 2,
  2: 3,
  length: 3
}
```

剩余参数：通过在函数参数上实现，以数组形式存储函数多余的参数；

```
function fn(a,b,...arr){
     arr[0];//为3
}
fn(1,2,3,4,5,6);
```

#### 立即执行函数

IIFE（Immediately Invoked Function Expression）= 函数表达式 + 立即调用；可以用来临时创建独立作用域，避免变量污染全局；

- 具名 IIFE 的函数名仅在函数内部有效，外部无法访问或调用，但可以在函数体内部用于递归调用；
- IIFE 属于函数表达式，不会像函数声明那样发生提升，而是在代码执行到该位置时才创建并立即执行；
- IIFE 可以有返回值，其返回结果通常用于变量初始化或模块封装。

```
(function () {
  console.log('执行了');
})();

(() => {
  console.log('执行了');
})();
    
const result = (function () { // 42
  return 42;
})();
```

#### 箭头函数

箭头函数是没有自身执行上下文特性的函数，主要用于固定 `this` 并简化函数写法

- 没有 `arguments` 对象，但可以使用剩余参数获取参数；
-  不能作为构造函数，没有 `prototype` 属性，不能使用 `new` 调用，没有自己的`super`（能继承外面的）
- 箭头函数没有自己的 `this`，箭头函数的`this` 在创建时就确定了，继承外层作用域，之后就锁定了，不受调用方式影响，并且无法通过 `call / apply / bind` 改变 `this`指向；

```
const obj = {
   name: 'aaa';
   fn: () =>{
      console.log(this);
      //这个this指向window，因为obj没有this，那他就沿用更外一层的this，即指向window的this
   }
}

const obj = {
   name: 'aaa';
   fn1: function() {
      let fn2 = () =>{
          console.log(this);//这个this指向obj，因为fn2的this沿用fn1的，而fn1的this指向obj
      }
   }
}
```

#### 闭包

闭包是指函数与其声明时所在的词法环境的组合， 当函数在其词法作用域之外执行时，依然可以访问其创建时的词法环境中的变量；

- 当函数执行时，会创建对应的执行上下文及词法环境；正常情况下，函数执行结束后，其作用域中的变量在没有被引用时会被垃圾回收；
- 如果内部函数引用了外部函数的变量，并且该内部函数在外部仍然可访问，那么这些变量就不会被回收；因为内部函数的作用域链中仍然持有对外部词法环境的引用

满足闭包的三个条件：

- 存在内部函数和外部函数嵌套；
- 内部函数引用了外部函数的变量；
- 内部函数在外部作用域中被访问；

```
function outer() {
  let count = 0;

  return function () {
    count++;
    return count;
  };
}

const fn = outer();

fn(); // 1
fn(); // 2
//count 没有被销毁 → 因为被内部函数引用
```

### this

this 是执行上下文中的一个绑定，指向当前函数的调用者（this由函数调用时决定，箭头函数除外）；

绑定规则：优先级为new > 显式绑定 > 隐式绑定 > 默认绑定

- 默认绑定：当函数以普通函数调用的方式执行时，就会触发默认绑定，这是的this在非严格模式下指向`window`，严格模式指向`undefined`；

  ```
  function fn() {
    console.log(this);//window
  }
  
  fn(); // 默认绑定
  ```

- 隐式绑定：当函数作为对象的方法被调用时，`this` 会自动绑定到调用该方法的对象上；函数作为对象属性被赋值给其他变量后，调用时不再通过原对象触发，此时的this就会隐式丢失，根据调用指向对应的调用者；

  ```
  const obj = {
    a: 1,
    fn() {
      console.log(this);//obj
    },
  };
  obj.fn();
  
  //隐式丢失
  const obj = {
    a: 1,
    fn() {
      console.log(this.a);
    },
  };
  const fn = obj.fn; // 函数被“提取”出来
  fn(); // undefined或  window
  ```

- 显式绑定：通过函数的 `call / apply / bind` 方法，手动指定函数执行时的 `this` 指向；

  1. call：立即调用一次函数，逐个传递参数，返回函数原来的return值；不会永久改变函数的this指向，用于临时改变this的指向来调用函数；

     ```
     let obj = {age:18}
     function fn(x,y){
         console.log(this);
     }
     fn.call(obj,1,2);//this指向obj，x=1，y=2
     ```

  2. apply：会立即调用一次函数，参数为数组形式，返回函数原来的return值；不会永久改变函数的this指向，临时改变this来调用函数；

     ```
     let obj = {age:18}
     function fn(x,y){
         console.log(this);
     }
     fn.apply(obj,[1,2]);
     ```

  3. bind：不会立即调用函数，返回将this改变指向后的新函数；不会改变原函数的this指向，但是了返回一个this改变的新函数；bind的this是强绑定，后续无法通过call/apply修改，但是可以通过bind再次绑定；

     ```
     let obj = {age:18}
     function fn(x,y){
         console.log(this);
     }
     
     fn = fn.bind(obj,1,2);//再次赋值fn来使函数this永久指向obj
     ```

- new 绑定：当函数通过 `new` 关键字调用时，会触发 new 绑定，此时 `this` 指向新创建的实例对象；

```
function fn1() {
    console.log('fn1', this);
}
let obj = {
    fn1,
    fn2: () => { console.log('fn2', this) },//指向window，因为obj不形成执行上下文
    fn3: function () {
        console.log('fn3', this)//指向obj
        function fn4() {
            console.log('fn4', this);//指向window
        }
        let fn5 = () => {
            console.log('fn5', this);//指向obj
        }
        fn4();
        fn5();
    },
}
fn1();//指向widow
obj.fn1();//指向obj
obj.fn2();//window
obj.fn3();//指向obj
```

### 类

类是用于创建对象的模板，本质上是对构造函数 + 原型链的语法糖封装；

- 类本质上还是一个函数，`typeof Class`为`function`，只是写法更接近传统面向对象语言；
- 类体默认运行在严格模式下，不会有变量提升；
- 在类体中使用this来访问类本身的属性和方法；

类的创建：

- constructor：创建实例时自动调用，用于创建和初始化类的对象；

- 实例方法：在类体定义，可以通过实例来调用，并且会挂在 `Person.prototype` 上，所有实例共享；


- static：用于声明静态属性或方法，挂在类本身上，不会被实例继承，只能通过类本身访问；

```
//类声明
class Person{
    static type = "human";//静态方法
    age = 18; // 实例属性
    constructor(name, age=18) {
        this.Myname = name;
        this.Myage = age;
    }
    sayHi() {//实例方法
    console.log("hi");
  }
}

//类表达式声明
const Person = class {
    constructor(name, age) {
        this.name = name
        this.age = age
    }
}
const p1 = new Person("张三", 18);                                                                                                  
```

#### 访问修饰符

public：可以被类内部方法，子类，类外部访问方法；

protected：可以被类内部方法，子类访问，可以使用_修饰符表示protected；

private：只能被类内部通过this访问，可以使用#表示private；

```
class Person{
    #age;//#表示age是private的
    _name;//_表示name是protected的
    
    constructor(name, age) {
        this.Myname = name;
        this.Myage = age;
    }
}
let a =new Person('aaa', 18);
```

#### new的底层

- new会在堆中新建一个空对象；
- 先将新对象的`__proto__`指向构造函数的prototype对象；
- 再将构造函数的this绑定到这个新对象中；
- 然后执行一次构造函数的代码，为这个新对象添加属性和方法；
- 根据构造函数的返回值决定最终返回结果：
  1. 如果构造函数 return 一个对象，会覆盖 new 创建的对象，并且返回的对象不会继承构造函数的 prototype；
  2. 如果return的是基本数据类型则会被忽略，继续返回新对象；

new的优先级：成员访问 .   >   new（带括号的，如`new Foo()`）   >   函数调用()   >   new（不带括号的，如`new Foo`）

```
- new foo().getname()
  → (new foo()).getname()
  → 先创建实例，再调用方法

- new foo.getname()
  → new (foo.getname)()
  → new 的是 foo.getname 方法

- new foo.getname
  → new (foo.getname)//直接成员访问
```

#### prototype原型对象

如果方法定义在构造函数内部，每次 new 都会创建一份新的函数，造成内存浪费；

所以每个构造函数在创建时都会自动拥有一个 prototype 对象；该对象用于存放可以被所有实例共享的属性和方法；当通过 new 创建实例时，实例会通过原型链访问 prototype 上的内容，从而实现方法复用；

-  prototype 上的方法在被实例调用时，方法中的this 指向调用该方法的实例对象；


- prototype 对象默认包含 constructor 属性，指向构造函数本身；同时，prototype 作为一个普通对象，其内部的`__proto__`指向 Object.prototype，用于形成原型链；

```
function Person(name,age){
    this.name=name;
    this.age =age;
}
//追加共享的方法
Person.prototype.say =function(){
    console.log('Hello');
}

//一次性向原型对象添加多个方法
Person.prototype = {
    constructor: Person(),//重新指回原来的构造函数，直接赋值会丢失prototype原来的constructor
    say: function(){},
    sing: function(){}
}

let a = new Person('aaa',18);
let b = new Person('bbb',10);
//这时a.say() === b.say()
```

#### `__proto__`原型属性

所有通过字面量或new构造函数创建的对象（包括构造函数，普通函数，数组，prototype对象等）都有`__proto__`属性，它指向构造函数的prototype原型，所有实例都通过`__proto__`访问构造函数的prototype对象上的方法；

构造函数本身也是函数对象：

- 构造函数的 proto 指向 Function.prototype；
-  Function.prototype 作为一个对象，其 proto 指向 Object.prototype；
-  Object.prototype 的 proto 指向 null，表示原型链的终点；

```
let a = new Person('aaa',18);
a.__proto__=== Person.prototype;
```




#### constructor属性

每一个实例对象或构造函数的prototype都有constructor属性，该属性指向它们的构造函数；

```
Person.prototype.constructor === Person;//为true

let a = new Person('aaa',18);
a.constructor === Person//为true
```

#### 原型链

原型链是一种对象属性和方法的查找机制；当访问对象的属性或方法时：

1. 先在对象自身查找；
2. 如果不存在，则通过 proto（即 [[Prototype]]）查找其原型对象；
3. 再沿着原型对象的 proto 继续向上查找；
4. 直到查找到 Object.prototype；
5. 若仍未找到，则继续到 null，查找结束并返回 undefined；

#### instanceof查找

instanceof 用于判断一个对象是否是某个构造函数的实例；他会沿着对象的原型链向上查找，看构造函数的 prototype 是否出现在该原型链上；，如果存在则返回 true，否则返回 false；

和typeof区别：typeof：用于判断数据类型（偏基础类型），instanceof：用于判断对象的构造关系（基于原型链）；

```
function Foo() {}
const f = new Foo();

console.log(f instanceof Foo); // true
```

### 类继承

通过extend关键字实现，如果子类有constructor要用super（父类的构造函数）来继承父类的属性，并且super要写在constructor最顶部，如果没有constructor，子类会自动继承父类属性；对于父类的方法，子类会自动继承；

```
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} 发出声音。`);
  }
}
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // 顶部调用父类的构造函数
    this.breed = breed;
  }
  speak() {
    console.log(`${this.name} 汪汪叫。`);
  }
}

const dog = new Dog('旺财', '金毛');
dog.speak(); // 输出: 旺财 汪汪叫
```

#### 原型链继承

让子构造函数的prototype指向父构造函数的实例对象

缺点：创建子实例对象时无法向父构造函数传参；所有子实例对象共享父的引用属性

```
    <script>
        //Father构造函数
        function Father(){
            this.money = 1000
        }
        //Child的构造函数
        function Child(){

        }
        //将Child的原型赋值为Fahter实例对象，以实现继承
        Child.prototype = new Father();

        let Child1 = new Child();
        console.log(Child1.money)
    </script>
```

#### 盗用构造函数

又称“对象伪装”，“经典继承”，在子构造函数中通过调用父构造函数的call或apply将父的this指向子；相当于将父构造函数宏的属性和方法复制到子构造函数中；

缺点：子构造函数和子实例与父的原型对象无联系，无法继承父亲写在prototype上的方法；由于父类必须在构造函数中定义方法，通过盗用构造函数继承的方法本质上都变成了实例自己的方法，不是公共的方法，因此失去了复用性；

```
    <script>
        //Father构造函数
        function Father(money){
            this.money = money
        }
        //Child的构造函数
        function Child(money){
            //将Father构造函数的this指向Child，并传参
            Father.call(this,money);
        }
        let Child1 = new Child(10000);
        console.log(Child1.money)
    </script>
```

#### 组合继承

子先通过盗用构造函数继承父的属性，再通过原型链继承父的共享方法；

缺点：调用了两次父的构造函数

```
    <script>
        //Father构造函数
        function Father(money){
            this.money = money
        }
        Father.prototype.SayHi= function(){
            console.log('Hello')
        }
        
        //Child的构造函数
        function Child(money){
            Father.call(this,money);
        }
        Child.prototype = new Father();

        let Child1 = new Child(10000);
        console.log(Child1.money)
        Child1.SayHi();
    </script>
```

#### 原型式继承

object函数将原型量继承核心代码封装成一个函数，基于已有的对象创建新对象，创建完后新对象的`__protp__`指向prototype，再指向已有对象；在ES5新增了一个`Object.create()`方法将原型式继承规范化了；

缺点：不能传参；已有对象的引用数据类型会被新对象共享；

```
const object = function(o){//o是一个已有对象
     //创建临时构造函数
     function F(){}
     //将临时构造函数的原型对象指向o
     F.prototype = o
     //这个object函数的返回值是一个原型对象指向o的F实例对象
     return new F()
}
let Father = {
    money:10000,
}

let Child = object(Father);
console.log(Child.money)
```

#### 寄生式继承

在原型式继承基础上增强对象；

```
const object = function(o){
    //原型式继承创建一个临时clone对象
    let clone = Object.create(o);
    clone.sayHi = function(){
        console.log('Hello');
    }
    return clone;
}
```

#### 寄生式组合继承

通过盗用构造函数继承父的属性，通过混合寄生式继承和原型式继承来继承父原型对象中的方法

```
    <script>
        //Father构造函数
        function Father(money){
            this.money = money
        }
        Father.prototype.SayHi= function(){
            console.log('Hello')
        }
        //Child的构造函数
        function Child(money){
            //盗用构造函数继承父的属性并可传参
            Father.call(this,money);
        }
        //创建一个实现寄生式继承的函数,函数参数是引用
        function inherit(Father,Child){
            //创建一个临时存储Fatter的原型对象的对象
            let Fatherprototype = Object.create(Father.prototype);
            //将临时Fatherprototype的constructor指向Child，以便将临时Father赋值给Child的原型对象时的constructor不是Child构造函数
            Fatherprototype.constructor =Child;
            //将配置好的临时Fatherprototype赋值给Child的原型对象，实现了Child的原型对象指向Fahter，并且constructor指向Child构造函数
            Child.prototype = Fatherprototype;
        }
        inherit(Father,Child);
    </script>
```
