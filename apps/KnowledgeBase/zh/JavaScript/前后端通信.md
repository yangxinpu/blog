# Ajax

**AJAX** （Asynchronous JavaScript and XML，异步 JavaScript 和 XML），一种用于在不刷新整个页面的情况下，向服务器交换数据，并更新部分网页内容的技术；

**长轮询**：一种基于AJAX技术实现的实时通信模式；客户端通过 AJAX 向服务器发送请求，服务器收到请求后，不立即返回响应，而是保持一段时间的连接，若期间服务器有新数据需要推送，就立即响应数据，否则响应空数据，同时关闭当前连接；对于客户端收到响应后（无论是空响应还是数据响应），会立即发送新的 AJAX 请求，形成等待 - 响应 - 再等待的轮询循环；

**短轮询**：一种基于AJAX实现客户端主动向服务端获取实时数据的通信模式；客户端每隔固定时间（如 1 秒、5 秒）就向服务器发送一次 AJAX 请求（比如使用setInterval进行发送），服务器收到请求后立即返回响应（无论是否有新数据），客户端处理响应后，等待下一个时间间隔再重复请求；缺点：数据更新的延迟取决于轮询间隔。例如间隔 5 秒，若新消息在第 3 秒产生，客户端要等到第 5 秒才能获取，最多有 5 秒延迟、大部分时间服务器可能没有新数据，但客户端仍会按间隔发送请求，浪费网络带宽和服务器资源；

### 基本原理

通过JS创建 `XMLHttpRequest` 对象（或使用 `fetch`），配置请求信息（如请求方式 GET/POST、请求 URL、是否异步等），将发送异步请求到服务器，并监听服务器的响应状态（如成功、失败、超时等），接收服务器返回的数据（JSON/XML ），用JS解析数据，并通过 DOM 操作更新页面局部内容；

AJAX 请求会受浏览器同源策略限制，只能请求同域名、同端口、同协议的资源，跨域请求需服务器配置 CORS；

### 网络请求方式：

网络请求方式是客户端与服务器之间交换数据的规则和模式，核心基于网络协议（如 HTTP、HTTPS、WebSocket 等），并通过不同的 方法或策略实现数据的发送、获取、更新等操作；

GET：从服务器获取指定资源，不会修改服务器数据；请求参数附加在 URL 中（如 `https://api.example.com/users?id=1`），参数有长度限制（通常 2KB~8KB，取决于浏览器），浏览器会缓存 GET 请求的响应（可通过`Cache-Control`控制）；

POST：向服务器提交数据，通常用于创建新的资源数据；请求参数放在请求体（Body） 中，无长度限制，可传输大量数据（如 JSON、表单数据、文件）；无默认缓存，每次请求都会触发服务器处理；

PUT：向服务器发送数据，用于全量更新指定资源（如果资源不存在则创建）；参数通常放在请求体中，具有幂等性（多次执行相同的 PUT 请求，结果一致）；

DELETE：请求服务器删除指定资源；参数可放在 URL或请求体中，具有幂等性（多次删除同一资源，结果一致）；

PATCH：用于部分更新服务器资源；参数可放在 URL或请求体中，不一定幂等（可能多次部分更新后，对应资源结果不一样）；

HEAD：用于客户端不传输实际内容的情况下，获取服务器对某个资源的元数据（如资源是否存在、大小、最后修改时间等），类似 GET，但只返回响应头，不返回响应体；

OPTIONS：用于查询服务器支持的通信方式，让客户端在发送实际请求前，先了解服务器允许的 HTTP请求方式、跨域规则、支持的请求头格式等信息；

CONNECT：建立隧道连接（如 HTTPS 中用于 SSL 握手）；

### XMLHttpRequest

用于在不重新刷新页面的情况下，与服务器进行异步 HTTP 通信；基于事件回调；

```
const xhr = new XMLHttpRequest();

xhr.open('GET', 'http://127.0.0.1',true);

xhr.onload = function () {
    if (xhr.status === 200) {
        console.log('请求成功', xhr.responseText);
    }
}

xhr.send();
```

**xhr实例**

status：服务器返回的 HTTP 状态码（如 200 表示成功、404 表示资源不存在、500 表示服务器错误）；

statusText：与 HTTP 状态码对应的文本描述；

responseText：服务器返回文本格式的响应数据（适用于 JSON、HTML、文本等）；

responseXML：服务器返回XML格式响应数据（（需服务器返回 `Content-Type: text/xml`）；

timeout：设置请求超时时间（单位：毫秒），超时后触发 `ontimeout` 事件；

withCredentials：设置是否允许请求携带 Cookie 等凭证

open()：配置请求参数，但不发送请求；第一个参数为请求方法，第二个参数为请求url，第三个参数为是否异步发送请求；

```
xhr.open('GET', 'http://127.0.0.1',true);
```

send()：发起请求到服务器；参数为请求体数据（`GET` 方法无需传，`POST` 可传字符串、`FormData` 等）；

setRequestHeader()：设置请求头（如 `Content-Type`、`Authorization`），必须在 `open()` 之后、`send()` 之前调用；

```
xhr.setRequestHeader('Authorization', 'Bearer abc123');
```

abort()：中断当前请求；

getResponseHeader()：获取服务器返回的指定响应头；

```
xhr.getResponseHeader('content-type')
```

onlaod：请求成功后触发的回调；

```
  xhr.onload = function () {
    if (xhr.status === 200) {
      console.log('请求成功', xhr.responseText);
    }
  }
```

onerror：请求失败时触发的回调；HTTP 错误不会触发；

onabort：调用 `abort()` 中断请求时触发；

onprogress：接收响应体时持续触发；

ontimeout：请求超时时触发；

### Fetch

基于Promise的网络请求API；第一个参数为URL，第二个为请求的配置对象，返回一个Promise对象，返回的Promise对象仅当仅网络错误为reject，否则都为resolve（就算是HTTP 4xx/5xx 错误也不会直接 reject）；

请求成功（resolve）时返回给then的参数为一个对象：含有ok，status，statusText参数，通过调用json()`、`text()、blob()、formData()等方法可以将响应体解析，链式调用返回给下一个then；

链式调用第二个then，回调函数的参数收到解析为JSON或text的数据，可对此数据进行操作；

```
fetch('http://localhost',{method:'GET'}).then((response) =>{
    if(!response.ok)throw new Error(response.status);
    return response.json();
}).then((data)=>{
    console.log('获得数据',data);
}).catch((err)=>{
    console.error('获得数据失败',err);
})
```

### Fetch与XHR区别

**xhr**： 

兼容IE7+的浏览器；

基于事件监听处理异步结果；

原生有abort方法终止请求；

无法在响应体传输过程中分片读取，若要处理大文件，需服务器配合分片传输，客户端手动拼接；

默认携带cookie；

提供 `onprogress` 事件，自动触发进度更新，可直接获取已加载大小和 总大小；

**fetch**：

仅支持现代浏览器（Chrome 42+、Firefox 39+、Safari 10.1+）；

基于 Promise，支持 async/await，链式调用then；

需借助浏览器原生的 `AbortController` 生成 信号（`signal`），关联到请求后通过 `controller.abort()` 取消；

```
//创建控制器和信号
const controller = new AbortController();
const signal = controller.signal;

//将信号传入 fetch 配置
fetch('/api/long-task', { signal })
.then(res => res.json())
.then(data => console.log(data))
.catch(error => {
  // 取消请求会触发 AbortError
  if (error.name === 'AbortError') {
    console.log('请求已取消');
  } else {
    console.error('错误：', error);
  }
});


setTimeout(() => {
  controller.abort(); // 通过控制器取消请求
}, 3000);
```

`fetch` 的 `Response.body` 是原生 `ReadableStream` 对象，可逐段读取响应数据；

`fetch` 默认在跨域请求中不携带 Cookie，需手动设置 `credentials: 'include'` 才会携带（同源请求默认携带）；

需将响应体转为 `ReadableStream`，手动读取流数据并累加已加载大小，计算进度；

```
fetch('/api/large-file')
.then(res => {
  const contentLength = Number(res.headers.get('Content-Length')); // 总大小
  let loaded = 0; // 已加载大小

  //获取流读取器
  const reader = res.body.getReader();

  //递归读取流数据
  function readStream() {
    return reader.read().then(({ done, value }) => {
      if (done) {
        console.log('下载完成');
        return;
      }
      // 累加已加载大小，计算进度
      loaded += value.byteLength;
      const progress = (loaded / contentLength) * 100;
      console.log(`下载进度：${progress.toFixed(2)}%`);
      
      return readStream(); // 继续读取下一段
    });
  }

  return readStream();
})
.catch(error => console.error('下载失败：', error));
```

### Axios

基于promise的网络请求库，能自动将数据转换为JSON格式；

浏览器：

- 基于 `XMLHttpRequest` 封装；将 XHR 的 `onload`、`onerror`、`onabort` 等事件封装为 Promise 的 `resolve`/`reject`；
- 通过配置对象（如 `method`、`headers`、`timeout`）简化 XHR 的 `open()`、`setRequestHeader()`、`send()` 等分散操作；
- 请求时自动将数据转为 JSON 字符串，响应时自动解析 JSON 数据；统一处理网络错误、超时错误、HTTP 错误（4xx/5xx 状态码）；

Node.js 环境：

- 基于 http/https 模块的封装；
- 这部分封装同样遵循Promise化和统一接口原则，将 `http.request()` 等回调式 API 转换为 Promise，并对齐浏览器环境的配置参数（如 `method`、`headers`、`timeout`）；

**安装**：`pnpm i axios`

#### 发起请求：

```
import axios from "axios";

//Promise写法
axios({
    method: 'post',//post请求
    url: 'https://frontest.sipc115.com/admin/auth/login',
    data: {
        userName: adminData.userName,
        password: adminData.password,
    }
}).then(res=> {
    console.log(res);
})

//try-catch写法
try {
  const res = await axios({
    url: 'https://v1.hitokoto.cn/',
    method: 'get',
  })
  console.log(res)
} catch (error) {
  console.log(error)
}

//根据请求方法来写
axios.get('/api/user', {
  params: {
    id: 1
  }
}).then(res => {
  console.log(res.data)
})
```

#### 请求配置:

```
axios({
  url: '/api/user',
  method: 'get',
  baseURL: 'https://api.example.com',
  timeout: 5000,
  params: {},         // URL 查询参数 ?a=1
  data: {},           // 请求体（POST/PUT）
  headers: {
    Authorization: 'Bearer token'
  },
  withCredentials:true, //是否携带cookie
  responseType: 'json'
})
```

#### Axios 实例

用于统一请求配置，避免每次写 baseURL、headers；

```
const service = axios.create({
  baseURL: '/api',
  timeout: 10000
})

//发送GET请求
service.get('/user')
```

#### **拦截器:**	

**请求拦截器**：在发送请求之前执行，use接收两个回调函数（只会触发其中一个函数）：

第一个是发起请求前前执行，参数为请求配置；可以在此进行校验，添加 token，修改参数等，其返回值作为最终发起HTTP请求的配置；

第二个参数为第一个回调抛出异常或返回 Promise.reject 时才会执行的回调，其返回如果为正常值会传给下一个 Promise 链的 then，如果为`throw error/Promise.reject(error)`会传给catch；

```
axios.interceptors.request.use(
  (config) => {
    return config
  },
  (error) => {
    return Promise.reject(error)
  }
)
```

**响应拦截器**：在服务器返回数据之后，请求数据进行处理之前执行，use接收两个参数：响应函数和错误函数；可以在此统一处理业务状态码等；

```
axios.interceptors.request.use(
  (res) => {
    return res
  },
  (error) => {
    return Promise.reject(error)
  }
)
```



# SSE

Server-Sent Events（服务器发送事件），是一种基于 HTTP 协议的技术，用于实现服务器向客户端**单向**推送实时数据的通信模式；

服务器端需通过 HTTP 响应头声明 `Content-Type: text/event-stream` 即可开启 SSE 流，客户端通过原生 `EventSource` API 即可发起连接和接收数据；

SSE 采用HTTP长连接（`Connection: keep-alive`）机制，连接建立后不会立即关闭，服务器可通过该连接持续向客户端推送数据，直到连接被主动关闭或异常中断；

当连接因网络中断、服务器故障等原因断开时，`EventSource` 会自动尝试重新连接（默认重连间隔为 3 秒）；服务器可通过 `retry` 字段自定义重连间隔，重连时，客户端会通过 `Last-Event-ID` 请求头携带上一次接收的事件 ID，服务器可据此ID恢复数据传输；

服务器只能传输文本数据，不能直接支持二进制数据（可以转为base64）；

SSE对服务端传输的数据格式有明确规范：

`event`：可选，指定事件名（默认事件名为 `message`），客户端可通过事件名监听特定类型的消息；

`data`：必选，数据内容（文本格式，通常为 JSON 字符串），多行数据需每行以 `data:` 开头，最终以空行 `\n\n` 结束。

`id`：可选，事件唯一标识，用于重连时恢复数据（客户端会缓存该 ID，重连时通过 `Last-Event-ID` 发送给服务器）；

`retry`：可选，指定客户端重连间隔（毫秒），覆盖默认的 3 秒；

```
// 连接服务器的 SSE 端点
const eventSource = new EventSource('http://127.0.0.1:80');

// 监听默认事件
eventSource.onmessage = (event) => {
  console.log('收到数据：', event.data);
};

// 监听连接打开
eventSource.onopen = () => {
  console.log('SSE 连接已打开');
};
```

### eventSource实例

#### 属性：

**readyState**：表示当前连接状态，有 3 种值：

0（`EventSource.CONNECTING`）：正在连接；

1（`EventSource.OPEN`）：连接已打开，可接收数据；

2（`EventSource.CLOSED`）：连接已关闭；

**url**：返回初始化 `EventSource` 时使用的 SSE 服务器完整URL

**withCredentials**：指示是否在跨域请求中使用凭据（如 Cookies、HTTP 认证信息），默认false；

#### 方法：

close()：主动关闭 SSE 连接；

#### 事件

onopen：与服务器的 SSE 连接成功建立时触发（`readyState` 变为 `1`）；

onmessage：收到服务器发送的默认事件时触发（服务器未指定 `event` 字段时，默认事件名为 `message`）通过参数对象的data属性可以获得收到的数据；

onerror：连接发生错误时触发（如网络中断、服务器返回非 200 状态码），参数为错误对象；

```
eventSource.onmessage = (event) => {
  console.log('收到数据：', event);
};

// 监听连接打开
eventSource.onopen = () => {
  console.log('SSE 连接已打开');
};

// 监听连接出错
eventSource.onerror = (error) => {
  console.error('SSE 连接出错:', error);
};
```

# WebSocket

**消息推送的方式：**

WebSocket：客户端发送HTTP请求，包含`Upgrade: websocket`和`Connection: Upgrade`头部，服务器返回101状态码，表示协议升级成功，连接从HTTP转化为WebSocket，连接建立后，客户端和服务器可以随时互相发送消息，无需重复建立连接，保持长连接状态；

### 服务端

安装： `npm install ws`；

#### 创建WebSocket服务器

使用WebSocket.Server方法，参数为host，port，path（访问服务器路径,默认为/），verifyClient（获得并验证客户端信息的方法）

```
const ws = new WebSocket.Server({
    host: 'localhost',
    port: 3000,
    path: '/ws',//,这里服务器路径为localhost:3000/ws
    verifyClient: (info) => {
        console.log(info);
        return true;
    }
});
```

#### ws实例方法

**ws.on**：监听服务器的特定事件，第一个参数为事件名称（connection，close，error），第二个参数为回调函数；

connection回调的参数：WebSocket 实例对象，该对象代表与客户端建立的连接，通过这个对象，你可以与对应的客户端进发送，接收消息，关闭连接等；

WebSocket 实例对象的属性和方法：

client.send：向客户端发送数据；

client.close：关闭当前客户端的连接；

client.on：监听特定事件（open，close，error，message）；

```
ws.on('connection', (client) => {
    console.log('新客户端已连接');
    // 当收到客户端消息时触发
    client.on('message', (message) => {
        console.log(`收到消息: ${message}`);
    });

    // 当客户端断开连接时触发
    client.on('close', () => {
        console.log('客户端已断开连接');
    });

    // 发生错误时触发
    client.on('error', (error) => {
        console.error('WebSocket错误:', error);
    });
});
```

**ws.close**：关闭整个 WebSocket 服务器，终止所有当前连接并停止接收新连接；

```
ws.close();
```

**ws.clients**：获取所有当前连接的WebSocket实例集合（Set）；

```
ws.on('connection', (client) => {
//将收到的消息gu
    client.on('message', (message) => {
        console.log(`收到消息: ${message}`);
        ws.clients.forEach((clients) => {
            if (clients.readyState === WebSocket.OPEN) {
                clients.send(message);
            }
        });
    });
});
```

### 客户端

对于客户端，WebSocket是js内置的对象，用于在客户端和服务器之间建立持久的双向通信连接，实现实时数据传输；

#### **创建WebSocket对象：**

```
const socket = new WebSocket('ws://localhost:8080');
```

#### socket实例

**socket.send**：向服务器发送数据，数据可以为字符串、Blob、ArrayBuffer 或 ArrayBufferView；

**socket.close**：关闭当前 WebSocket 连接，第一个参数为状态码（如1000 表示正常关闭，默认 1005表示无状态码），第二个参数为关闭原因字符串；

```
socket.close(1000, "客户端主动关闭");
```

**socket上的事件**

```
socket.onopen = function() {
  console.log("连接已建立，可以开始通信");
};

socket.onmessage = function(event) {
  console.log("收到消息：", event.data);
  // 若消息是JSON格式，可解析为对象
  const data = JSON.parse(event.data);
};

socket.onclose = function(event) {
  console.log(`连接关闭（状态码：${event.code}，原因：${event.reason}`);
};

socket.onerror = function(error) {
  console.error("WebSocket错误：", error);
};
```
