# Functions & OOP

### Functions

Functions are essentially objects; can be assigned, passed as parameters, returned as values, have properties attached;

#### arguments and Rest Parameters

arguments: Built-in pseudo-array inside function, stores all parameters passed to function;

```js
function fn() {
  console.log(arguments);
}
fn(1, 2, 3);

// Output
{
  0: 1,
  1: 2,
  2: 3,
  length: 3
}
```

Rest parameters: Implemented via function parameters, stores extra parameters as array;

```js
function fn(a, b, ...arr) {
     arr[0]; // Is 3
}
fn(1, 2, 3, 4, 5, 6);
```

#### IIFE (Immediately Invoked Function Expression)

IIFE = Function Expression + Immediate Invocation; can be used to temporarily create independent scope, avoiding variable pollution of global;

- Named IIFE's function name is only valid inside function, cannot be accessed or called externally, but can be used for recursive calls inside function body;
- IIFE is a function expression, doesn't hoist like function declarations; created and immediately executed when code reaches that position;
- IIFE can have return values, usually used for variable initialization or module encapsulation.

```js
(function () {
  console.log('Executed');
})();

(() => {
  console.log('Executed');
})();
    
const result = (function () { // 42
  return 42;
})();
```

#### Arrow Functions

Arrow functions are functions without their own execution context, mainly used to fix `this` and simplify function writing;

- No `arguments` object, but can use rest parameters to get parameters;
- Cannot be used as constructor, no `prototype` property, cannot use `new` call, no own `super` (can inherit from outside);
- Arrow functions don't have own `this`; arrow function's `this` is determined at creation, inherits outer scope, then locks, unaffected by call method, and cannot change `this` binding via `call / apply / bind`;

```js
const obj = {
   name: 'aaa';
   fn: () => {
      console.log(this);
      // this points to window, because obj doesn't have this, so it uses outer this, pointing to window
   }
}

const obj = {
   name: 'aaa';
   fn1: function() {
      let fn2 = () => {
          console.log(this); // this points to obj, because fn2's this inherits fn1's, and fn1's this points to obj
      }
   }
}
```

#### Closures

Closure refers to the combination of a function and its lexical environment at declaration time; when a function executes outside its lexical scope, it can still access variables in its creation-time lexical environment;

- When function executes, corresponding execution context and lexical environment are created; normally, after function execution ends, variables in its scope are garbage collected if not referenced;
- If inner function references outer function's variables, and this inner function is still accessible externally, these variables won't be collected; because inner function's scope chain still holds reference to outer lexical environment

Three conditions for closure:

- Inner function and outer function nesting exists;
- Inner function references outer function's variables;
- Inner function is accessed in outer scope;

```js
function outer() {
  let count = 0;

  return function () {
    count++;
    return count;
  };
}

const fn = outer();

fn(); // 1
fn(); // 2
// count not destroyed → because referenced by inner function
```

### this

this is a binding in execution context, pointing to current function's caller (this determined at function call time, except for arrow functions);

Binding rules: Priority is new > explicit binding > implicit binding > default binding

- **Default binding**: When function executes as ordinary function call, default binding is triggered; `this` points to `window` in non-strict mode, `undefined` in strict mode;

```js
function fn() {
  console.log(this); // window
}

fn(); // Default binding
```

- **Implicit binding**: When function is called as object's method, `this` automatically binds to the object calling the method; after function is assigned to other variable as object property, calling no longer triggers through original object, `this` implicitly lost, pointing to corresponding caller according to call;

```js
const obj = {
  a: 1,
  fn() {
    console.log(this); // obj
  },
};
obj.fn();

// Implicit loss
const obj = {
  a: 1,
  fn() {
    console.log(this.a);
  },
};
const fn = obj.fn; // Function "extracted"
fn(); // undefined or window
```

- **Explicit binding**: Manually specify function's `this` binding at execution time via function's `call / apply / bind` methods;
  1. call: Immediately calls function once, passes parameters individually, returns function's original return value; doesn't permanently change function's this binding, used to temporarily change this to call function;
  2. apply: Immediately calls function once, parameters as array, returns function's original return value; doesn't permanently change function's this binding, temporarily changes this to call function;
  3. bind: Doesn't immediately call function, returns new function with changed this binding; doesn't change original function's this binding, but returns a new function with changed this; bind's this is strong binding, cannot be modified via call/apply later, but can bind again via bind;

```js
let obj = {age: 18}
function fn(x, y) {
    console.log(this);
}
fn.call(obj, 1, 2);  // this points to obj, x=1, y=2
fn.apply(obj, [1, 2]);

fn = fn.bind(obj, 1, 2); // Reassign fn to make function's this permanently point to obj
```

- **new binding**: When function is called via `new` keyword, new binding is triggered, `this` points to newly created instance object;

```js
function fn1() {
    console.log('fn1', this);
}
let obj = {
    fn1,
    fn2: () => { console.log('fn2', this) },      // Points to window, because obj doesn't form execution context
    fn3: function () {
        console.log('fn3', this)                   // Points to obj
        function fn4() {
            console.log('fn4', this);              // Points to window
        }
        let fn5 = () => {
            console.log('fn5', this);              // Points to obj
        }
        fn4();
        fn5();
    },
}
fn1();       // Points to window
obj.fn1();   // Points to obj
obj.fn2();   // window
obj.fn3();   // Points to obj
```

### Classes

Class is a template for creating objects, essentially syntactic sugar wrapping constructor + prototype chain;

- Class is still a function, `typeof Class` is `function`, just syntax closer to traditional OOP languages;
- Class body runs in strict mode by default, no variable hoisting;
- Use this inside class body to access class's own properties and methods;

Class creation:

- constructor: Automatically called when creating instance, used to create and initialize class's object;
- Instance methods: Defined in class body, can be called via instance, mounted on `Person.prototype`, shared by all instances;
- static: Declares static properties or methods, mounted on class itself, not inherited by instances, can only be accessed via class itself;

```js
// Class declaration
class Person {
    static type = "human";  // Static method
    age = 18;               // Instance property
    constructor(name, age = 18) {
        this.Myname = name;
        this.Myage = age;
    }
    sayHi() {               // Instance method
        console.log("hi");
    }
}

// Class expression declaration
const Person = class {
    constructor(name, age) {
        this.name = name
        this.age = age
    }
}
const p1 = new Person("Zhang San", 18);
```

#### Access Modifiers

public: Can be accessed by class internal methods, subclasses, class external methods;

protected: Can be accessed by class internal methods, subclasses; can use _ modifier to indicate protected;

private: Can only be accessed internally via this; can use # to indicate private;

```js
class Person {
    #age;    // # indicates age is private
    _name;   // _ indicates name is protected
    
    constructor(name, age) {
        this.Myname = name;
        this.Myage = age;
    }
}
let a = new Person('aaa', 18);
```

#### new's Underlying Mechanism

- new creates a new empty object in heap;
- First sets new object's `__proto__` to constructor's prototype object;
- Then binds constructor's this to this new object;
- Then executes constructor's code once, adding properties and methods to this new object;
- Determines final return result based on constructor's return value:
  1. If constructor returns an object, it overrides the object created by new, and returned object doesn't inherit constructor's prototype;
  2. If returns primitive type, it's ignored, continues returning new object;

new's priority: Member access . > new (with parentheses, like `new Foo()`) > Function call () > new (without parentheses, like `new Foo`)

```js
- new foo().getname()
  → (new foo()).getname()
  → Create instance first, then call method

- new foo.getname()
  → new (foo.getname)()
  → new is foo.getname method

- new foo.getname
  → new (foo.getname) // Direct member access
```

#### prototype Object

If methods are defined inside constructor, each new creates a new copy of the function, wasting memory;

So each constructor automatically has a prototype object when created; this object stores properties and methods that can be shared by all instances; when creating instances via new, instances access prototype's content via prototype chain, achieving method reuse;

- Methods on prototype, when called by instance, have this pointing to the instance object calling the method;
- prototype object by default contains constructor property, pointing to constructor itself; meanwhile, prototype as a plain object, its internal `__proto__` points to Object.prototype, for forming prototype chain;

```js
function Person(name, age) {
    this.name = name;
    this.age = age;
}
// Add shared methods
Person.prototype.say = function() {
    console.log('Hello');
}

// Add multiple methods to prototype object at once
Person.prototype = {
    constructor: Person(), // Point back to original constructor, direct assignment loses prototype's original constructor
    say: function() {},
    sing: function() {}
}

let a = new Person('aaa', 18);
let b = new Person('bbb', 10);
// Now a.say() === b.say()
```

#### `__proto__` Prototype Property

All objects created via literal or new constructor (including constructors, ordinary functions, arrays, prototype objects, etc.) have `__proto__` property, it points to constructor's prototype; all instances access constructor's prototype object's methods via `__proto__`;

Constructor itself is also a function object:

- Constructor's proto points to Function.prototype;
- Function.prototype as an object, its proto points to Object.prototype;
- Object.prototype's proto points to null, indicating prototype chain's end;

```js
let a = new Person('aaa', 18);
a.__proto__ === Person.prototype;
```

#### constructor Property

Every instance object or constructor's prototype has constructor property, this property points to their constructor;

```js
Person.prototype.constructor === Person; // true

let a = new Person('aaa', 18);
a.constructor === Person // true
```

#### Prototype Chain

Prototype chain is a lookup mechanism for object properties and methods; when accessing object's property or method:

1. First look in object itself;
2. If not exists, look in its prototype object via proto (i.e., [[Prototype]]);
3. Then continue looking up via prototype object's proto;
4. Until reaching Object.prototype;
5. If still not found, continue to null, lookup ends and returns undefined;

#### instanceof Lookup

instanceof is used to determine if an object is an instance of a constructor; it looks up along object's prototype chain to see if constructor's prototype appears on that prototype chain; if exists returns true, otherwise false;

Difference from typeof: typeof: Used to determine data type (basic types), instanceof: Used to determine object's constructor relationship (based on prototype chain);

```js
function Foo() {}
const f = new Foo();

console.log(f instanceof Foo); // true
```

### Class Inheritance

Implemented via extend keyword; if subclass has constructor, use super (parent's constructor) to inherit parent's properties, and super must be at top of constructor; if no constructor, subclass automatically inherits parent's properties; for parent's methods, subclass automatically inherits;

```js
class Animal {
  constructor(name) {
    this.name = name;
  }
  speak() {
    console.log(`${this.name} makes a sound.`);
  }
}
class Dog extends Animal {
  constructor(name, breed) {
    super(name); // Call parent's constructor at top
    this.breed = breed;
  }
  speak() {
    console.log(`${this.name} barks.`);
  }
}

const dog = new Dog('Wang Cai', 'Golden Retriever');
dog.speak(); // Output: Wang Cai barks.
```

#### Prototype Chain Inheritance

Make child constructor's prototype point to parent constructor's instance object

Disadvantages: Cannot pass parameters to parent constructor when creating child instance objects; all child instance objects share parent's reference properties

```js
// Father constructor
function Father() {
    this.money = 1000
}
// Child constructor
function Child() {

}
// Set Child's prototype to Father instance object to implement inheritance
Child.prototype = new Father();

let Child1 = new Child();
console.log(Child1.money)
```

#### Constructor Stealing

Also called "object masquerading", "classic inheritance"; in child constructor, call parent constructor's call or apply to point parent's this to child; equivalent to copying parent constructor's properties and methods to child constructor;

Disadvantages: Child constructor and child instance have no connection with parent's prototype object, cannot inherit methods written on parent's prototype; since parent class must define methods in constructor, methods inherited via constructor stealing essentially become instance's own methods, not shared methods, thus losing reusability;

```js
// Father constructor
function Father(money) {
    this.money = money
}
// Child constructor
function Child(money) {
    // Point Father constructor's this to Child, pass parameters
    Father.call(this, money);
}
let Child1 = new Child(10000);
console.log(Child1.money)
```

#### Combination Inheritance

Child first inherits parent's properties via constructor stealing, then inherits parent's shared methods via prototype chain;

Disadvantages: Calls parent's constructor twice

```js
// Father constructor
function Father(money) {
    this.money = money
}
Father.prototype.SayHi = function() {
    console.log('Hello')
}

// Child constructor
function Child(money) {
    Father.call(this, money);
}
Child.prototype = new Father();

let Child1 = new Child(10000);
console.log(Child1.money)
Child1.SayHi();
```

#### Prototypal Inheritance

Object function encapsulates prototypal inheritance core code, creates new object based on existing object; after creation, new object's `__proto__` points to prototype, then to existing object; ES5 added `Object.create()` method to standardize prototypal inheritance;

Disadvantages: Cannot pass parameters; existing object's reference data types are shared by new objects;

```js
const object = function(o) { // o is an existing object
     // Create temporary constructor
     function F() {}
     // Point temporary constructor's prototype object to o
     F.prototype = o
     // This object function's return value is an F instance object whose prototype object points to o
     return new F()
}
let Father = {
    money: 10000,
}

let Child = object(Father);
console.log(Child.money)
```

#### Parasitic Inheritance

Enhances object on basis of prototypal inheritance;

```js
const object = function(o) {
    // Prototypal inheritance creates temporary clone object
    let clone = Object.create(o);
    clone.sayHi = function() {
        console.log('Hello');
    }
    return clone;
}
```

#### Parasitic Combination Inheritance

Inherits parent's properties via constructor stealing, inherits methods in parent prototype object via combining parasitic inheritance and prototypal inheritance

```js
// Father constructor
function Father(money) {
    this.money = money
}
Father.prototype.SayHi = function() {
    console.log('Hello')
}
// Child constructor
function Child(money) {
    // Constructor stealing inherits parent's properties and can pass parameters
    Father.call(this, money);
}
// Create function implementing parasitic inheritance, function parameter is reference
function inherit(Father, Child) {
    // Create temporary object storing Father's prototype object
    let Fatherprototype = Object.create(Father.prototype);
    // Point temporary Fatherprototype's constructor to Child, so when assigning temporary Father to Child's prototype object, constructor is Child constructor
    Fatherprototype.constructor = Child;
    // Assign configured temporary Fatherprototype to Child's prototype object, implements Child's prototype object pointing to Father, and constructor points to Child constructor
    Child.prototype = Fatherprototype;
}
inherit(Father, Child);
```
